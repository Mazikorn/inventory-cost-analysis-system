### 7.3 已知局限

1. **静态页面数据隔离**：每个 HTML 页面拥有独立的内存数据（`suppliersData`、`categoryData`、`consumableData`、`inboundData`），跨页面数据不共享。因此库存联动（inbound → inventory-list）无法在原型环境中端到端验证，需在集成测试环境完成。
2. **编码自动生成**：当前验证方式为表单提交 + JS 数据注入 + 表格渲染验证，真实后端自动生成逻辑需在 API 集成测试中确认。
3. **库存总价值计算**：`20×500 + 10×520 = 15,200` 为理论值，需在后端集成环境验证实际计算结果。
